1. Backend и логика (Flask & Python)

Сердце сайта написано на Flask. В приложении реализована маршрутизация (routes) для регистрации, логина и главной страницы.

    requirements.txt: Все зависимости (Flask, Flask-SQLAlchemy, Flask-Login) вынесены в отдельный файл для быстрой развертывания окружения.

2. Работа с данными (ORM & DB)

Используется SQLAlchemy ORM, что позволяет работать с базой данных через Python-классы (модели).

    Модели данных: Созданы таблицы User (пользователи), Item (предметы) и Requirement (ресурсы).

    Связи: Реализована связь «Один ко многим» между предметом и его ресурсами (db.relationship).

    Хранение: Все данные хранятся в файле barter.db (SQLite), что делает проект портативным.

3. Авторизация и файлы

    Регистрация и вход: Реализованы через Flask-Login. Пароли хранятся в защищенном виде (хэшируются через werkzeug.security).

    Загрузка файлов: Пользователи могут загружать свои аватарки. Реализована проверка расширений файлов и сохранение под уникальными именами (через secure_filename).

4. Интерфейс (Frontend & Шаблонизация)

    Шаблоны (Jinja2): Интерфейс разделен на переиспользуемые блоки. Данные о пушках передаются из Python-списка прямо в HTML-цикл, что позволяет легко добавлять новые предметы в items_data.py.

    Bootstrap: Используется для оформления форм, навигационной панели и модальных окон с ресурсами.

    Визуализация дерева: С помощью кастомного CSS реализована система «коннекторов» (линий), которые динамически соединяют карточки оружия в ветки.

5. API и сторонние данные
